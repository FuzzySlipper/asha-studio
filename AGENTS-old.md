# ASHA Studio agent guidance

`asha-studio` is the distinct frontend-heavy public consumer for ASHA Studio workflows.

## Boundary rules

- `boundary-policy.json` is the machine-readable source for `pnpm run check:boundaries`.
- Use public ASHA package roots only.
- Allowed in source now: `@asha/command-registry`, `@asha/contracts`, `@asha/editor-tools`.
- Local package-manager links must match the allowed public package roots in `boundary-policy.json`.
- Deferred later public packages: `@asha/studio-evidence`, `@asha/runtime-bridge`, `@asha/devtools`, `@asha/renderer-three`.
- Do not import ASHA internals, generated files by path, native/raw transports, engine Rust paths, package `src/**` subpaths, or aliases into `/home/dev/asha`.
- Do not add arbitrary JSON command hatches such as `call(methodName, json)` or private UI-only mutation callbacks.
- GUI and agent surfaces must use the same command identities and visible timeline/readout model.
- If the studio needs a missing capability, create/request a public ASHA surface rather than bypassing the boundary.
- Compatibility metadata lives in `src/compatibility.ts`; update it with tests and the sample session artifact whenever ASHA public compatibility markers change.
- Session/workspace timeline logic lives in `src/session-workspace.ts`; GUI and agent-originated commands must continue through the same `invokeStudioCommand` path and exported readout fixture.
- Voxel workflow logic lives in `src/voxel-workflow.ts`; preview must stay editor-local while apply uses typed public `VoxelCommand` proposal/apply evidence and records before/after hashes.
- Visual evidence/review export logic lives in `src/visual-evidence.ts`; exports must fail closed when timeline/results, visual refs, render deltas, sequence correlation, or compatibility evidence are missing/stale.
- V1 proof command logic lives in `scripts/v1-proof.ts` with readback validation in `scripts/readback-v1-proof.mjs`; generated `artifacts/` output is reproducible and git-ignored.
- Batch/undo workflow logic lives in `src/command-batch.ts`; batches must stay bounded to known public command ids, record atomic/best-effort transaction mode, per-command result shape, retry/undo metadata, and failure classification, and may derive explicit typed inverse commands for V1 rather than inventing a generic authority undo stack.
- Model/material preview logic lives in `src/model-material-preview.ts`; it may use `@asha/contracts` package-root DTOs for catalog/material/static-mesh/render-diff metadata, but Studio must still not import runtime transports directly until `@asha/runtime-bridge` compatibility is approved here.
- Viewport editor panel logic lives in `src/viewport-editor-panel.ts`; it must project selected target, preview-vs-applied state, timeline correlation, and evidence refs from the shared workspace model rather than owning authority or using a private UI-only path.
- Entity browser logic lives in `src/entity-browser.ts`; it is a deterministic projection of `StudioSceneViewModel` renderables (entity ids, labels, asset/material badges, source-state badges, selected state) and must not become a private ECS/product asset DB. Hierarchy selection syncs to the viewport selected renderable through the public `selection.set_active_entity` command on the shared timeline (no private UI-only callback). It fails closed with classified diagnostics on hierarchy/readback drift, missing selected entity, stale entity-list hash, or unsupported/private entity sources. `pnpm run proof:entity-browser` regenerates `artifacts/entity-browser-proof/` with `scripts/readback-entity-browser-proof.mjs`; `fixtures/studio-entity-browser.sample.json` is reproducible from the workspace model.
- Selected entity inspector logic lives in `src/selected-entity-inspector.ts`; it is a deterministic projection of the shared scene-view selected entity (identity/provenance/read-only transform plus a typed field list) and must not become a private ECS component store. Exactly one field (name) is editable, and the edit must flow through the public `entity.set_name` command on the shared timeline (no private UI-only mutation callback or freeform JSON field write); the applied name updates the viewport 3D readback (`selectedEntityName` + `selected-entity-name:` marker) and the exported agent readout. It fails closed with classified diagnostics on `missing_selected_entity`, `inspector_readback_drift`, `stale_inspector_state`, `unsupported_field_edit`, and `edit_command_mismatch`. Transform/material/provenance fields stay read-only browser/reference readback. `pnpm run proof:inspector` regenerates `artifacts/selected-entity-inspector-proof/` with `scripts/readback-selected-entity-inspector-proof.mjs`; `fixtures/studio-selected-entity-inspector.sample.json` is reproducible from the workspace model.
- Selection/transform gizmo logic lives in `src/transform-gizmo.ts`; it is a deterministic projection of the shared scene-view selected entity into a transform gizmo read model (selection, three visible axis handles with one command-wired active axis, and a single-axis translate with independently-computed before/after evidence) and must not become a private ECS transform store. The translate previews editor-locally and commits through the new public `transform.translate_entity` command on the shared timeline (preview + apply as distinct sequences) — no generic event bus, no private UI-only manipulation callback, and no freeform JSON transform write; the viewport 3D readback (`buildStudioViewport3dReadback` gizmo arg) and host render the selection outline, axis handles, preview ghost, applied box, and a `transform_gizmo` readback with `gizmo-translate` markers. It fails closed with classified diagnostics on `missing_selected_entity`, `stale_gizmo_selection`, `missing_gizmo_handle`, `transform_readback_mismatch`, and `private_mutation_path`. Only single-axis translate is supported in this slice (rotate/scale/multi-axis deferred); it makes no physics/native/Agora/GPU/performance claims and authoritative transform mutation stays behind the runtime-bridge gate. `pnpm run proof:gizmo` regenerates `artifacts/transform-gizmo-proof/` with `scripts/readback-transform-gizmo-proof.mjs`; `fixtures/studio-transform-gizmo.sample.json` is reproducible from the workspace model.
- Demo asset loading logic lives in `src/demo-asset-loading.ts`; it builds the demo asset package (Catalog/AssetLock/FlatSceneDocument over `@asha/contracts` DTOs), loads it through the public `scene.load_asset` command identity into the shared timeline, emits a retained-mode render diff plus named renderable ids and provenance, projects the loaded asset through the existing Three.js viewport host, and must fail closed (never silently render fallback geometry) on missing asset, unsupported format, material mismatch, or stale catalog/lock drift, classifying each failure with public catalog/lock/scene validation codes. `pnpm run proof:asset-load` regenerates `artifacts/asset-load-proof/` with `scripts/readback-asset-load-proof.mjs` validation; refresh browser-captured proofs (`proof:browser`/`proof:visual-capability`) in a Chromium environment to recapture the viewport with the loaded asset.
- Browser visual capture logic lives in `scripts/browser-visual-capture.ts` with readback validation in `scripts/readback-browser-capture.mjs`; it must wait for proof-content markers, correlate with the V1 proof artifact/timeline/hashes, record screenshot SHA-256 values, and classify Chromium browser screenshots without claiming Agora/GPU/performance evidence.
- Agora compositor capture proof logic lives in `src/agora-compositor-capture.ts` (pure validation model) and `scripts/agora-compositor-capture.ts` (live harness); it is an OPTIONAL, additive visual proof backend. The harness serves the built `dist/`, launches it as a real webview surface under the Wayfire/Agora compositor (`compositorctl launch`), lets the launched surface settle, and captures the compositor surface (`compositorctl capture --export`). The `agora_compositor_capture_proof` classifies its path explicitly (`captureBackend: agora_compositor`, `captureMode: compositor_surface`, plus the compositor-reported `compositorCaptureBackend`), verifies a nonblank frame via BOTH the compositor `visual_inspection` status and an independent ImageMagick metric, checks surface identity/dimensions and timestamp-based frame freshness, and correlates the frame (stamped `asha_command_sequence_id`) with the studio `render.capture_before_after` timeline sequence and the optional `studio_visual_capability_proof`. Frame presentation is proved by `visual_inspection: visible` plus the independent pixel metrics; `frameCountAtCapture`/`frame_count` is informational only because WebKitGTK webview surfaces can render visible content without incrementing it. It preserves `not_hardware_gpu`/`not_performance_evidence`/`not_native_runtime`/`not_wasm_authority` non-claims (a compositor frame is not GPU/performance evidence) and fails closed with classified negative smokes on missing/blank/wrong/stale capture and uncorrelated timeline. It is environment-gated (requires a live compositor) and is NOT part of `pnpm run verify`; `studio_visual_capability_proof` references it as an optional backend (`optionalBackends`) but never requires it, so the existing browser proof stays unweakened. `pnpm run proof:agora-compositor` regenerates `artifacts/agora-compositor/` with `scripts/readback-agora-compositor-capture.mjs`; that artifact is live (timestamps/sha of a real frame) and git-ignored, so unit coverage uses a synthetic evidence fixture in `test/agora-compositor-capture.test.ts`. See `agora-os/agent-guide-connect-to-agora-shell-for-tests` for the compositor stack.
- Native runtime mode is approved only through the `@asha/runtime-bridge` package root with `runtime-bridge.v0` metadata and the `pnpm run proof:runtime-bridge` proof/readback path; raw `@asha/native-bridge` and `@asha/wasm-replay-bridge` imports remain forbidden.
